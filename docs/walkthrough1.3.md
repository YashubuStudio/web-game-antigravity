# Neon Bumper v1.1 - 拡張機能 実装完了レポート

## 実装した新機能 (v1.2 & Mobile Update & Polish)
- **パフォーマンス最適化と描画の滑らかさ**:
  - サーバーの物理更新・通信頻度、およびクライアントからの入力送信頻度を30Hz(秒間30回)にしていますが、クライアント側で**線形補間（Lerp）処理**を入れることで、30Hzでもカクつくことなく60Hz相当の滑らかなフレームレート(rAF)で描画されるように改善しました。
  - セッション参加上限を10名に拡大しました。
  - タイトル画面にてリアルタイムで「現在プレイ中」と「待機中（観戦者）」の人数が確認できるようになりました。
- **モバイル対応とアスペクト比固定**:
  - 画面サイズに応じてUIが自動的に縮小・最適化されるレスポンシブデザインを適用しました。また、マップが縦長に潰れるCSS・Canvasの不具合を修正しました。
  - タッチ操作に対応しました（スワイプで移動箇所指定、画面タップでダッシュ）。
- **ゲームバランスと視覚表現の強化**:
  - **HP数値の復活**: HPバーの上に残りのHP（数値）が表示されるように修正しました。
  - **ダッシュ無敵時間の短縮**: ダッシュ時の完全無敵時間（iFrames）を 300ms から **150ms** に半減させました。今までより当たり判定が早く復活し、多段ヒットによるデスが増えるためゲームに緊張感が出ます。
  - **ダッシュ状態の視覚化**: プレイヤーの見た目がダッシュのCT(クールダウン)状態に応じて3段階に変化するようになりました。
    1. **攻撃中(無敵)**: 白く発光し、枠線が太くなります。
    2. **CT待機中(使用不可)**: 全体が半透明に暗くなり、オーラが消えます。
    3. **ダッシュ可能(Ready)**: 通常のネオンカラーとオーラに戻ります。
- **NPC（Bot）の自動追加とリソース削減**:
  - マップ上の「人間プレイヤー」が1名以上かつ合計プレイヤーが3名以下の状態が30秒間続いた場合のみ、合計人数が5名になるまで自動的に「Bot」がスポーンします。
  - **リソース最適化**: サーバー内に人間プレイヤーが「0人」になった瞬間に、すべてのBotを自動的に削除し、サーバーの演算リソースを節約するよう改良しました。
  - Botは一番近い人間（または他のBot）をゆっくりと追いかけ、一定距離まで近づくとたまにダッシュ攻撃を試みますが、人間プレイヤーよりも弱く設定されているため「スコア（キル）稼ぎ」として活用可能です。

## 実装した新機能 (v1.1)
- **ゲームルールの拡張**:
  - フィールドの半径を1.7倍に拡大。
  - 1セッションの参加上限を6人(現在は10人)に設定。上限以降は自動的に対象セッションの「観戦モード」になります。
  - スポーン時、他のプレイヤーからできるだけ離れた安全な位置からスタートするロジックを実装。
- **キル（ラストヒット）システム**:
  - ダッシュで衝突した相手に「ラストヒットマーカー」が付与されます。
  - その相手が外周ペナルティまたは衝突によってHPが0になった際、最後にヒットを与えた生存プレイヤーの「キル数」が加算されます。
  - キル発生時、画面中央にキルログが数秒間表示されます。
- **描画とHUD表現の強化**:
  - プレイヤーのキル数に応じて自身の形状が進化します（0=円、1〜3=三角、4=四角、5以上=角数が増加するポリゴン）。
  - 1キル以上持っているプレイヤーは発光オーラを纏います。
  - ゲーム内で最もキル数（または同率ならHP）が高いトッププレイヤーの頭上に王冠「👑」が付きます。
  - 右上にアルファ30%のミニマップを追加。
  - 右下にダッシュのクールダウンタイマーを「X.X 秒」単位で表示。
  - ゲーム参加前のメニューで自身のネオンカラーをカラーピッカーから自由に選べるようになりました。名前被りも問題なく動作します。
- **ダッシュ中の無敵化とリスポーン**:
  - ダッシュ中は完全無敵となり、被ダメージが0になります。
  - 死亡時（観戦モード時）専用の「YOU DIED」画面を表示。5秒経過するとシームレスに再参加（Re-spawn）ボタンが出現します。
- **データ最適化**:
  - サーバーとクライアント間でやり取りする浮動小数点データを小数点第2位で丸める処理を入れ、通信の軽量化を図りました。
