## 提案する変更 (Node.js/Game側)

### [server.js](file:///c:/Users/canva/OneDrive/Desktop/test-code/web-game/server.js) (ゲームロジック・通信)
*   **フィールドと人数制限**: `ARENA_RADIUS` を 1700 (1000 * 1.7) に拡大。最大参加人数（生存者）を6人に制限。上限到達時の接続者は自動的に「観戦モード」になります。
*   **スポーンロジック強化**: 新規参加時、他の既存プレイヤーから最も離れた安全な座標を計算してスポーンするように変更します（名前はハッシュやランダムIDを内部で付与し、重複を許容します）。
*   **キル/ラストヒット判定**:
    *   プレイヤーオブジェクトに `kills` と `lastHitBy` (最後にダメージを与えてきたプレイヤーID)のプロパティを追加します。
    *   ダッシュ衝突時に相手の `lastHitBy` を更新します。
    *   プレイヤーのHPが0になった際（外周ダメージ含む）、`lastHitBy` のプレイヤーが生存していれば、そのプレイヤーの `kills` を +1 します。
*   **ダッシュ無敵化**: ダッシュ中（`vx`, `vy`が一定以上、またはフラグ管理）の他プレイヤーからの被ダメージ、および外周ダメージを0にします。
*   **データ軽量化**: ソケットで送信する座標や速度などの数値を `Math.round(val * 100) / 100` で小数点第2位に丸めて通信を最適化します。
*   **外部API連携**: プレイヤーの死亡時、`kills >= 5` の条件を満たせば、外部のPHPサーバーのURL（設定値）へ HTTP POST を行いスコアを送信します。
*   **リスポーン**: 死亡後5秒経過すると、画面上で「再参加」ボタンが表示され、シームレスに（名前・色を引き継いで）再参戦できるイベントを追加します。

### [public/game.js](file:///c:/Users/canva/OneDrive/Desktop/test-code/web-game/public/game.js) & [public/style.css](file:///c:/Users/canva/OneDrive/Desktop/test-code/web-game/public/style.css) (フロントエンド表示)
*   **参加前UIの強化**: 色（ネオンカラー）を選択できるカラーピッカー UIを追加します。
*   **形状と王冠の描画 (Canvas)**:
    *   キル数に応じた形状描画 (0:円、1〜3:三角、4:四角、n≧5:n角形)
    *   1キル以上のプレイヤーへのアウトラインオーラ（グロー効果）の強化
    *   現在のセッション内で1位（キル数またはHP）のプレイヤーの頭上に王冠アイコン（Canvasのパス描画またはテキスト）を描画
*   **UI HUDの追加**:
    *   **右下(UI)**: ダッシュアクションのクールダウンまでの時間を「X.X 秒」と小数点第1位まで表示。
    *   **右上(Canvas)**: アルファ値(透過)30%のミニマップを描画。全体のアリーナと各プレイヤーの点を縮小描画します。
    *   **左上(UI)**: 現在のサーバー内（セッション内）ランキングと、外部サーバーから取得したトップランキング結果（名前とキル数）を表示します。

## 外部サーバー (PHP + SQL) 設計案
ゲームサーバーとは分離し、以下のファイルを [.md](file:///c:/Users/canva/OneDrive/Desktop/test-code/GEMINI.md) アーティファクトとして作成・提供します。
*   **`save_score.php`**: Node.jsサーバーから受け取ったスコア（名前、キル数、到達日時）をバリデーションしてデータベースに `INSERT` するAPI。
*   **`get_ranking.php`**: データベースからキル数の高い順にトップランキングを `SELECT` してJSONで返すAPI（フロントエンドの左上に表示するため）。
*   **`schema.sql`**: 上記システムに必要なMySQL（MariaDB）のテーブル作成用のクエリ（DDL）。
