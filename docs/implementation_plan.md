# Neon Bumper (ネオン・バンパー) - 開発プラン

ブラウザで遊べる、軽量かつ初見でも分かりやすいマルチプレイヤー型のアリーナ対戦ゲーム「Neon Bumper」を提案します。

## ゲーム概要
*   **プレイヤー**: ネオンカラーに光る円形のキャラクターを操作します。
*   **目的**: アリーナ内で他のプレイヤーに体当たり（ダッシュ）して、外側のダメージゾーンに押し出す（または一定のダメージを与える）ことで最後まで生き残ることを目指します。
*   **操作**: マウスカーソルの方向へ自動またはクリックで移動。クリックで「ダッシュ（体当たり）」。
*   **特徴**: 
    *   軽量：重い画像アセットを使わず、HTML5 Canvasの描画（円や光るエフェクト）のみで表現します。
    *   マルチプレイ：Node.js + Socket.io を使い、WebSocket経由で2〜3人（またはそれ以上）がリアルタイムに同じアリーナで対戦できます。

## ユーザーレビューが必要な項目
> [!NOTE]
> ゲームの方向性（スマブラ風の押し出しサバイバルゲーム）について、こちらのアイデアで進めてよろしいでしょうか？
> もし「もっとパズル要素が欲しい」「協力プレイが良い」などのご要望があれば、プランを調整いたします。

> [!IMPORTANT]
> サーバーはNode.jsを使用します。（PHPはリアルタイム同期に向かないため、Node.js + Socket.ioの構成とさせてください）
> 開発は `web-game` ディレクトリ上で行います。

## 提案する変更

### `web-game` （新規構築）

#### [NEW] `package.json`
Node.jsプロジェクトの設定ファイル。`express` と `socket.io` を依存関係として追加します。

#### [NEW] `server.js`
ゲームサーバー。
*   静的ファイルの配信 (Express)
*   プレイヤーの接続・切断管理 (Socket.io)
*   **サーバー主導のゲームループ**: 60FPSでプレイヤーの位置、速度、衝突判定、アリーナの縮小などを計算し、全クライアントにブロードキャストします。

#### [NEW] `public/index.html`
フロントエンドのエントリーポイント。
*   フルスクリーンの `<canvas>` 要素
*   名前入力と「ゲーム開始」ボタンを含むシンプルなUI

#### [NEW] `public/style.css`
ダークテーマ・ネオン調の軽量なCSS。

#### [NEW] `public/game.js`
フロントエンドのゲームロジック。
*   サーバーからのGameStateの受信と補間（スムーズな描画）
*   キャンバスへの描画（プレイヤーの円、軌跡、アリーナの境界線）
*   マウス入力の取得とサーバーへの送信（現在のマウス座標等）

## 検証計画

### 手動検証 (Manual Verification)
1.  **サーバー起動**: `web-game` ディレクトリで `npm install` 後、`node server.js` を実行。
2.  **複数クライアントの接続**: ブラウザで `http://localhost:3000` を2つ以上のタブ（または別のブラウザ）で開く。
3.  **操作と同期テスト**:
    *   各タブで名前を入力してゲームに参加。
    *   マウス操作でキャラクターが動き、他のタブでも遅延なくスムーズにその動きが反映されるか確認。
4.  **衝突とゲームルールのテスト**:
    *   プレイヤー同士を衝突させ、弾き飛ばされる物理挙動が適切か確認。
    *   アリーナ外に出た際にダメージを受け、HPが0になるとゲームオーバー（観戦モード）になるか確認。
