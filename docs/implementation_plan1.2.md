# Neon Bumper v1.2 - パフォーマンス調整・NPC・待機表示 実装計画

追加のご要望に基づき、ゲームのパフォーマンス調整と、少人数時のNPC追加システム、そして待機人数表示機能を実装するためのプランを作成しました。

## 提案する変更

### [server.js](file:///c:/Users/canva/OneDrive/Desktop/test-code/web-game/server.js) 
*   **通信および物理の軽量化**: 
    *   サーバーのループ処理とクライアントへの送信レートを **60Hzから30Hz (`1000/30`)** に変更します。これに伴い、フレーム間の移動量が倍になるため、各種挙動（物理パラメータ）が極端に変わらない程度の微調整を行います。
*   **人数上限の変更**: `MAX_PLAYERS` を 6 から **10** に引き上げます。
*   **ロビー情報送信**: 
    *   現在の「プレイ中の人数」と「待機・観戦中の人数」を計算し、定期的に（またはプレイヤー接続時に）送信する機能を追加します。
*   **NPC(ボット)システムの追加**:
    *   3秒ごとに人数をチェックするタイマーを設けます。
    *   「現在マップ上にいるプレイヤー（PC＋NPC含む）」が3人以下の場合、**30秒間のカウントダウン**を開始します。
    *   30秒経過しても人間プレイヤーが新しく参加せず3人以下のままであった場合、マップ上の人数が **5人** になるまでランダムなNPCエンティティをスポーンさせます。
    *   **NPCのAIループ**: 
        *   一番近い他の生存プレイヤー（またはNPC）を探してターゲットにします。
        *   ターゲットに向けてゆっくり移動します（通常のプレイヤーより速度も加速度も低く設定）。
        *   一定距離内にターゲットがいれば一定確率でダッシュを試みますが、クールダウンを長めにし、倒しやすい強さに調整します。
    *   人間プレイヤーが上限に達した場合など、NPCが不要になった場合は死亡させるかキックするロジックを挟みます（今回はキルカウントの餌として機能するようにするため、HP0で消滅するPCと同じ扱いにします）。

### [public/index.html](file:///c:/Users/canva/OneDrive/Desktop/test-code/web-game/public/index.html) & [public/game.js](file:///c:/Users/canva/OneDrive/Desktop/test-code/web-game/public/game.js)
*   **参加前UI（スタート画面）の改善**:
    *   サーバーから受け取った「現在プレイ中: X人 / 待機中(観戦): Y人」という情報を表示する要素を [index.html](file:///c:/Users/canva/OneDrive/Desktop/test-code/web-game/public/index.html) に追加します。
*   **描画更新**:
    *   サーバーが30Hzになったため、クライアント側でコマ落ち（カクつき）を感じにくくするための「補間（Interpolation）」を強化したいところですが、今回は要件通り「物理と送信を両方30Hz」へシンプルに変更し、様子を見ます。
